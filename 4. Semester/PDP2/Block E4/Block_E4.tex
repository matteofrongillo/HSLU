\part*{Block E4 - Effective prompting}
\section*{How to write a good prompt (30 min)}
\subsection*{Rules to follow}
\begin{itemize}
    \item Start with the simplest request
    \item Use clear separators between different tasks, such as\\[.5ex]
        \texttt{\#\# INTRODUCTION}\\
        \texttt{\#\# OUTPUT STYLE}
    \item Use clear words to tell the AI what you want, for instance ``Translate'', ``List'', ``Explain''
    \item Be very specific as you add details: ``Explain the difference between renewable and non-renewable energy to a 16-year-old. Use simple words. Include three examples each.''
    \item Go to the point and remove all the unnecessary information.
    \item Do not be generic and specify what you want: ``\sout{Write a few paragraphs}''.\\
        Better: ``Write 2 paragraphs'' or ``Write a 200 word essay''
    \item Instead of saying what not to do, just say what to do.
\end{itemize}

\subsection*{Good prompt example}
\begin{verbatim}
    You are a librarian

    ## RULES
    Before starting generating the output, ask:
    - How old are you?
    - What is your favorite book?
    - How many books per year do you read?
    - What is your favorite book genre?

    ## GOAL
    Suggest a book based on the user's age, favorite book, number of books read the past year,
    and book genre.

    ## OUTPUT
    - Return a list of 5 books
    - Add a 30 word description for each book
    - Ask the user if they want 5 more books suggested
    - Do not ask further questions

    ## CONSTRAINTS
    - Do not suggest the same book
    - Do not suggest books from the same saga, if any
\end{verbatim}

\subsection*{Bad prompt example}
\begin{verbatim}
Suggest some Tolkien-style books.
\end{verbatim}

\section*{Exercise (15 min)}
Write a good prompt following the requirements:
\begin{itemize}
    \item The output must be an Arduino \texttt{.ino} file
    \item The code must use the \texttt{Seeed\_AMG8833\_driver.h} library
    \item The hardware is an AMG8833 thermal sensor with an 8x8 matrix, for a total of 64 pixels
    \item The program must read the temperature values detected by all 64 pixels
    \item The program must determine the highest temperature among the 64 pixels
    \item The highest detected temperature must be printed in degree Celsius as output
\end{itemize}

\newpage
\subsection*{Possible exercise solution}
\begin{verbatim}
You are an Arduino programmer

## GOAL
Write Arduino code that reads temperature data from an AMG8833 thermal sensor
and prints the highest detected temperature.

## HARDWARE
The hardware is an AMG8833 thermal sensor.
The sensor provides an 8x8 thermal matrix, for a total of 64 pixels.

## REQUIREMENTS
- The code must use the Seeed_AMG8833_driver.h library
- The code must read the temperature values from all 64 pixels
- The code must determine the highest temperature among the 64 pixels
- The code must print the highest detected temperature in degree Celsius

## OUTPUT
- Return only Arduino .ino code
- The code must be ready to compile
- Do not include explanations outside the code

## CONSTRAINTS
- Do not print all 64 pixel values
- Do not use a different library
- Do not add unnecessary features
\end{verbatim}

\subsection*{Possible output}
\begin{lstlisting}[language=C++]
#include <Seeed_AMG8833_driver.h>

AMG8833 sensor;
float pixels[64];

void setup() {
  Serial.begin(115200);

  if (sensor.init() != 0) {
    Serial.println("AMG8833 init failed");
    while (1) {
      delay(1000);
    }
  }
}

void loop() {
  sensor.read_pixel_temperature(pixels);
  float maxTemp = pixels[0];
  for (int i = 1; i < 64; i++) {
    if (pixels[i] > maxTemp) {
      maxTemp = pixels[i];
    }
  }

  Serial.print("Highest temperature: ");
  Serial.print(maxTemp, 2);
  Serial.println(" °C");
  delay(500);
}
\end{lstlisting}
